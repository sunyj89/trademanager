# 分期范围说明（基于《客户管理系统建设方案_V1_grace》）

## 一期（已覆盖）
- 线索（Lead）
- 客户（Customer）
- 联系人（Contact）
- 跟进（Follow-up）
- 代理商（Agent）
- 商机（Opportunity）

## 二期（已覆盖核心）
- 合同（Contract）
- 回款（Payment）

## 本轮推进
- 增加线索转化 API（Lead -> Customer/Opportunity），并拦截重复转化
- 增加商机赢单 API（Opportunity -> Contract），并拦截重复赢单
- 增加利润汇总报表 API（基于合同+回款聚合）
- 增加销售漏斗报表 API（按商机阶段聚合）

## 当前落地能力
- 业务对象新增/列表 + 分页筛选
- 关键关联校验与唯一冲突处理（409）
- 转化动作支持业务链路状态推进
- SQLite 持久化与 OpenAPI 自动文档

## 后续建议
- 增加佣金结算与销售目标达成分析
- 增加多币种利润换算口径与财务对账视图
- 引入 Alembic 管理数据迁移
- 增加鉴权、数据权限、操作日志


## 本轮补强
- 补充合同-商机-客户归属一致性约束
- 补充取消合同禁止录入回款约束
- 新增客户价值汇总报表（合同/回款/利润按客户聚合）

- 新增代理佣金汇总报表（按代理聚合合同额与预计佣金）

- 新增商机丢单动作与丢单原因留存
- 新增合同状态流转规则（完成态不可回退）
- 新增合同回款进度报表（合同额/已回款/未回款/进度）

- 新增合同超额回款拦截规则
- 新增合同完成需全额回款规则
- 新增合同状态汇总报表（按状态聚合数量和金额）

- 新增 MVP 可视化页面（根路径 `/`），用于快速查看经营指标与基础列表
